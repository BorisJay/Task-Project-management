# Task-Project-management
Task and project management system


**Скорочений варіант опису структури та функціоналу проекту**

Структура та функціонал системи менеджменту задачами та проектами, пов'язаними з ремонтом, обслуговуванням та виготовленням різноманітного заводського обладнання, з елементами складського обліку, з підтримкою веб-інтерфейсу та можливістю керування через Telegram-бот. 


**Основні Модулі Системи:**

1. **Користувачі та Ролі**
2. **Обладнання**
3. **Задачі**
4. **Проекти**
5. **Складський облік**
6. **Повідомлення та Сповіщення**
7. **Звіти та Аналітика**
8. **Налаштування**


**Деталізація Функціоналу по Модулях:**

**1. Користувачі та Ролі:**

- **Функціонал:**
    - Реєстрація та автентифікація користувачів.
    - Система ролей з різними рівнями доступу (Адміністратор, Менеджер проекту, Технічний спеціаліст, Комірник, Переглядач).
    - Керування профілями користувачів (ПІБ, контактна інформація, спеціалізація).
    - Призначення користувачів на задачі та проекти.


**2. Обладнання:**

- **Функціонал:**
    - Додавання, редагування, видалення інформації про одиниці обладнання.
    - Поля для обладнання: ID, Інвентарний номер, серійний номер, Назва, Тип обладнання, Виробник, Модель, Розташування (цех, ділянка, пов'язане обладнання), Відповідальна особа, Історія обслуговування/ремонтів, Технічна документація (прикріплені файли).
    - Можливість зв'язування обладнання з задачами та проектами.
    - Пошук та фільтрація обладнання.
    - документація та технічні паспорти


**3. Задачі:**

- **Функціонал:**
    - Створення первинних заявок.
    - Створення задач на підставі заявок, редагування, видалення.
    - Типи задач: Ремонт, Обслуговування (планове/позапланове), Виготовлення деталі/вузла, Модернізація, Діагностика, Інше.
    - Поля задачі: Назва, Опис, Тип задачі, Пріоритет (Високий, Середній, Низький), Статус (Нова, В роботі, Очікує запчастини, Виконана, Скасована), Відповідальний виконавець, Спостерігачі, Пов'язане обладнання, Дедлайн, Плановані трудовитрати, Фактичні трудовитрати, Список необхідних матеріалів/запчастин зі складу, Коментарі та історія змін.
    - Можливість прив'язки задач до проектів.
    - Можливість створювати нові задачі на підставі виконаних задач
    - Чек-листи або підлеглі задачі в рамках задачі.
    - Прикріплення файлів (фото пошкоджень, схеми тощо).
    - Пошук та фільтрація задач за різними критеріями.


**4. Проекти:**

- **Функціонал:**
    - Створення, редагування, видалення проектів.
    - Поля проекту: Назва, Опис, Менеджер проекту, Статус (Планується, Активний, Завершений, Призупинений), Дата початку, Планована дата завершення, Фактична дата завершення, Бюджет (опціонально), Список пов'язаних задач, Список залученого обладнання, Учасники проекту.
    - Візуалізація проекту.
    - Відстеження прогресу проекту на основі статусу пов'язаних задач.


**5. Складський облік:**

- **Функціонал:**
    - Додавання, редагування, видалення номенклатури матеріалів/запчастин.
    - Поля номенклатури: Артикул/Код, Назва, Одиниця виміру, Категорія, Поточний залишок, Мінімальний/Незнижуваний залишок, Постачальник (опціонально), Вартість (опціонально), Місце зберігання (склад, секція), Зображення, креслення, специфікація.
    - Операції: Надходження на склад, Видача зі складу (з прив'язкою до задачі/проекту), Списання, Інвентаризація.
    - Автоматичне списання матеріалів при виконанні задачі (за попередньо затвердженим списком).
    - **Формування заявки на замовлення матеріалів або запчастин виконується через модуль заявок**
    - Сповіщення про низький залишок матеріалів.
    - Історія рухів по кожній позиції номенклатури.
    - Пошук та фільтрація по складу.


**6. Повідомлення та Сповіщення:**

- **Функціонал:**
    - Сповіщення користувачів про:
        - Призначення на задачу/проект.
        - Зміну статусу задачі/проекту, де користувач є відповідальним або спостерігачем.
        - Нові коментарі до задачі/проекту.
        - Наближення дедлайну задачі.
        - Низький залишок матеріалів на складі.
    - Надсилання сповіщень через веб-інтерфейс (можливо, з використанням WebSocket для push-сповіщень) та Telegram.
    - Налаштування користувачами типів сповіщень, які вони хочуть отримувати.


**7. Звіти та Аналітика:**

- **Функціонал:**
    - Звіти по статусах задач (кількість нових, в роботі, виконаних).
    - Звіти по виконаних задачах за період (з трудовитратами).
    - Звіти по використанню матеріалів зі складу (з прив'язкою до задач/проектів).
    - Звіти по ефективності виконавців (кількість виконаних задач, середній час виконання).
    - Статистика по ремонтах обладнання (які одиниці ламаються найчастіше).
    - Можливість експорту звітів (CSV, Excel).


**8. Налаштування:**

- **Функціонал:**
    - Керування довідниками (типи обладнання, типи задач, одиниці виміру, категорії матеріалів, тощо).
    - Налаштування порогових значень для сповіщень про залишки на складі.
    - Керування користувачами та ролями.


**Telegram Bot Інтеграція:**

- **Функціонал:**
    - Авторизація користувача Telegram в системі.
    - Отримання сповіщень з системи.
    - Швидке створення заявки.
    - Швидке створення нової задачі (з мінімальним набором полів) на підставі незакритих заявок.
    - Отримання списку призначених задач.
    - Запит статусу конкретної задачі за її номером/назвою.
    - Додавання коментарів до задачі.
    - Запит інформації про залишок матеріалу на складі.


**Веб-інтерфейс:**

- **Функціонал:**
    - Дашборд з ключовими показниками (кількість активних задач, прострочені задачі, стан складу тощо).
    - Зручні форми для створення та редагування всіх сутностей (обладнання, задачі, проекти, номенклатура).
    - Табличні представлення даних з можливістю сортування, фільтрації та пошуку.
    - Сторінки деталей для кожної сутності з усією пов'язаною інформацією.
    - Календарне представлення задач та дедлайнів.
    - Інтерфейс для перегляду та формування звітів.
    - Розділ налаштувань.
